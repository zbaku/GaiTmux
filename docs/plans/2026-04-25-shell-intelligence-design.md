# Shell Intelligence Design - AI 驱动的终端智能化

> Created: 2026-04-25
> Status: Draft
> Priority: P0 (优先于远程连接功能)

## 概述

为 AIShell 添加 Shell 智能化功能，采用 **本地优先 + AI 解释** 的混合方案。

**核心功能**：
- 语法高亮（本地 Rust 实现）
- 自动补全（Fish-shell 风格）
- 智能提示（本地命令数据库）
- 命令解释（AI 驱动，用户按 `?` 键触发）

**设计原则**：
- **本地处理**：语法高亮、自动补全全部在 Rust 端完成，无网络延迟
- **AI 解释**：用户主动触发，按 `?` 键调 AI 解析当前命令
- **性能优先**：输入时实时解析，每帧 < 1ms

## 1. 语法高亮

### 实现方式：Rust 端实时解析

```rust
// 终端输入行的 token 类型
enum ShellToken {
    Command(String),      // 命令
    Option(String),       // 选项
    Path(String),         // 路径
    String(String),       // 字符串
    Pipe(String),         // 管道
    Variable(String),     // 变量
    Comment(String),      // 注释
}
```

### 颜色方案

| Token 类型 | 颜色 | ANSI 代码 | 示例 |
|-----------|------|-----------|------|
| 命令 | 绿色 | `\e[32m` | `ls` `git` `adb` |
| 选项 | 蓝色 | `\e[34m` | `-l` `--help` |
| 路径 | 青色 | `\e[36m` | `/sdcard/` `./file` |
| 字符串 | 黄色 | `\e[33m` | `"hello"` `'world'` |
| 管道 | 灰色 | `\e[90m` | `\|` `>` `>>` |
| 变量 | 洋红 | `\e[35m` | `$HOME` `$(pwd)` |
| 注释 | 暗灰 | `\e[37m` | `# 注释` |

### 性能要求

- 输入时实时解析
- 每帧渲染 < 1ms
- 支持多字节字符（中文）

## 2. 自动补全

### Fish-shell 风格的本地补全引擎

```
用户输入: ls /sdcard/Do
                    │
                    ▼
┌─────────────────────────────────────┐
│         补全引擎匹配流程            │
├─────────────────────────────────────┤
│  1. 识别上下文：命令=ls, 参数=路径  │
│  2. 扫描目录：/sdcard/              │
│  3. 过滤匹配：Do*                   │
│  4. 返回候选项                      │
└─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────┐
│  候选项（底部弹出）：               │
│  ┌─────────────────────────────┐   │
│  │ 📁 Documents/               │   │
│  │ 📁 Download/                │   │
│  │ 📁 Downloads/               │   │
│  └─────────────────────────────┘   │
│  Tab 下一项  Enter 确认  Esc 取消  │
└─────────────────────────────────────┘
```

### 补全来源

| 类型 | 来源 | 示例 |
|------|------|------|
| 命令 | `$PATH` 扫描 + 内置列表 | `git`, `adb`, `fastboot` |
| 文件路径 | 实时扫描目录 | `/sdcard/Download/` |
| 选项 | 命令选项数据库 | `git --com<tab>` → `--commit` |
| 历史 | 命令历史记录 | 之前输入过的命令 |

### 交互设计

- **Tab**: 切换下一个候选项
- **Shift+Tab**: 切换上一个候选项
- **Enter**: 确认选中项
- **Esc**: 取消补全
- **继续输入**: 自动过滤候选项

## 3. AI 命令解释

### 触发方式

- 光标在命令行时按 `?` 键
- AI 流式输出解释
- 包含风险评估 + 参数详解

### UI 设计

```
用户输入: find /sdcard -name "*.jpg" -mtime +30 -exec rm {} \;
用户按:  ?

┌─────────────────────────────────────────────────────────┐
│  AI 解释（流式输出，可折叠）：                          │
│                                                         │
│  📋 命令解释：                                          │
│  在 /sdcard 目录下查找所有 jpg 文件，                   │
│  且修改时间超过 30 天，然后删除这些文件。               │
│                                                         │
│  ⚠️ 风险评估：高风险操作                                │
│  - 会永久删除文件                                       │
│  - 建议先执行 find ... 不带 -exec rm 查看结果           │
│                                                         │
│  📖 参数详解：                                          │
│  -name "*.jpg"  匹配 jpg 文件                          │
│  -mtime +30     修改时间超过 30 天                     │
│  -exec rm {}    对每个匹配项执行 rm                    │
│                                                         │
│  [复制命令] [安全版本] [关闭]                          │
└─────────────────────────────────────────────────────────┘
```

### Token 优化策略

- 只发送当前命令行内容
- 使用精简的 system prompt
- 默认使用快速模型（如 deepseek-chat）
- 流式输出，用户可随时关闭

### System Prompt

```
你是一个 Linux 命令解释助手。用户会输入一个命令，你需要：
1. 用简洁的中文解释命令的作用
2. 如果有风险，明确警告
3. 解释每个参数的含义
4. 如果有更安全的替代方案，提供建议

回复格式：
📋 命令解释：[一句话总结]
⚠️ 风险评估：[无/低/中/高] + [说明]
📖 参数详解：[逐个参数解释]
```

## 4. 文件结构

### Rust 模块

```

├── highlight/
│   ├── mod.rs           # 高亮模块入口
```
